Запити Зтягнення (Pull Requests)

Часто ви робите зміни і покращення до проекте, що зкопіювали, ви захочете відправити ці зміни до супровідника оригіналу і request (запитати/попросити) щоб вони pull (зтягнули) ці зміни в оригінал, щоб будь-хто міг отримати переваги від змін, які в pull request (запиті зтягнення).

Ми хочемо створити список тих, хто завершив навчання, тому зробіть pull request аби додати ваш файл із ім'ям користувача до оригіналу: github.com/jlord/patchwork.

Створіть запит зтягнення

Відвідайте оригінальний репозиторій, що ви зкопіювали, на GitHub. В даному випадку - http://github.com/jlord/patchwork.

Часто GitHub помітить, що ви надслали зміни у гілку в зкопійованому репозиторії і покаже інформацію про це згори сайту. Якщо ви бачите, що це гілка 'add-username', то можете створити Pull Request звідти, якщо ж ні:

  • Натисніть зелену кнопку 'New pull request'.
  • Оберіть гілку із змінами, що ви хочете надіслати. Це повинна бути гілка названа 'add-yourusername'.

Тепер ви побачите сторінку із деталями вашого запиту зтягнення, яки в процесі збереження. Ця сторінка показує всі коміти та зміни у вигляді різниці між вашим запитом зтягнення у порівнянні із гілкою оригіналу 'gh-pages'.

Якщо оригінал містить документацію по співпрацюванюю, GitHub дасть посилання на неї. Це документація від власника репозиторію щодо правильності внесення змін до проекту — дуже корисна інформація, якщо ви хочете, аби ваші зміни прийняли!

Якщо все на сторінці відповідає вашим очікуванням:

  • Натисніть 'Create pull request'
  • Додайте назву та опис до ваших змін, що ви хочете, щоб автор зтягнув
  • Натисніть 'Send pull request'!

Бінго! Ви створили запит зтягнення — насолодіться цією миттю ;)

Якщо все чудово із вашим запитом зтягнення, він буде злитий впродовж хвилини. Якщо він не був злитий автоматично, впродовж декількох хвилин, ви, швидше за все, отримаєте якісь коментарі від @reporobot із поясненнями чому він не злив їх. Якщо це так, відмініть ваш запит, зробіть відповідні зміни, надішліть їх до вашої копії та створіть новий запит зтягення (це повідомить @reporobot про необхідність нової перевірки).
