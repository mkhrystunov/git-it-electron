Створіть новий файл у вашому репозиторії і додайте певний текст до цього файлу, потім створіть коміт із цим файлом.

Коміти

Коміти є ключовими у викорістанні Git. Вони це моменти, в яких ви зберігаєте і описуєте зроблену роботу. Вони - це тіки на часовій шкалі історії вашого проекту.

Створіть новий файл

Тепер, коли ви маєте Git репозиторій додайте до нього файл.

Відкрийте ваш текстовий редактор і створіть пустий файл. Тепер напишіть в ньому якийсь текст, можливо, ви надрукуєте "Hello world!" і збережіть під назвою "readme.txt" у папці 'hello-world', яку ви створили у попередньому завданні.

Перевірте Статус + Додайте зміни і створіть Коміт

Далі перевірте status(статус) вашого репозиторію, щоб з'ясувати чи відбулися із ним певні зміни. Ви знаєте, що так, а от чи знає Git?

Переконайтеся, що ви досі знаходитеся в папці 'hello-world' коли виконуватимете наступні команди. Використовуйте Git для перевірки, що змінилося у вашому репозиторії:

Спершу перевірте статус:

git status

Git повинен сказати вам, що було додано файл.

Тепер add(додайте) файл, що ви щойно створили аби він став частиною змін, які ви закомітите (збережете) із Git:

git add readme.txt

Нарешті, commmit("закомітьте") ці зміни до історії репозиторію із коротким повідомленням (message - m), що описує ваші зміни.

git commit -m "Created readme"

Крок: Зробіть Більше Змін

Тепер додайте новий рядок до 'readme.txt' і збережіть файл знову.

В терміналі ви можете переглянути differrence (разницю) між файлом тепер і яким він був у вашому останньому коміті.

Скажіть Git'у показати вамdiff(різницю):

git diff

Тепер, знаючи попередній крок, зробіть коміт із останніх змін.
