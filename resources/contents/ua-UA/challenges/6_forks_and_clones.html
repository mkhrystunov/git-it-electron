Зкопіюйте проект на GitHub.com і клонуйте його локально.

Копії

До сих пір, ви створили проект локально і надіслали його на GitHub, але це лише половина веселощів! Інша половина це робота із іншими людьми та іншими проектами.

Коли ви fork (копіюєте) репозиторій, ви створюєте його копію на своєму GitHub акаунті. Ваша копія починає своє диття в якості віддалного репозиторію — вона існує лише на вашому GitHub акаунті, а не на вашому комп'ютері. Копії використовуються задля створення вашої власної версії проекту (це зміна оригіналу, як взявши інший поворот на дорозі) або задля створення власного внеску в існуючий проект (наприклад, виправлення помилок або додавання нового функціоналу).

Щоб отримати зкопійований репозиторій із вашого GitHub акаунту на ваш комп'ютер ви clone (клонуєте) його. Це клонування створить копію віддаленого репозиторію на вашому комп'ютері, так що ви зможете працювати із ним локально.

Зкопіюйте репозиторій Patchwork

Проект, із яким ми працюватимемо, знаходиться за адресою github.com/jlord/patchwork. Перейдіть на його сторінку та натисніть кнопку 'Fork' згори зправа. Як тільки анімація копіювання завершиться, ви матимете копію на вашому акаунті. Зкопіюйте адресу щойно зкопійованого репозиторію із адресного рядку вашого браузеру, це адреса вашої копії на серверах GitHub.

Клонуйте вашу копію локально

Тепер у терміналі ви клонуєте репозиторій на ваш комп'ютер. Це автоматично створить нову папку для нього, тож немає потреби створювати її самому. Але переконайтеся, що ви не клонуєте, будучи всередині іншого Git репозиторію!

Переконайтеся, що ви не знаходитеся в папці 'hello-world' із попередніх завдань. Щоб вийти із папки, аби ви могли клонувати репозиторій, виконайте наступні дії:

Вийти із папки 'hello-world':
Підказка: дві крапки позначають папку на один рівень вище

cd ..

Тепер, коли ви більше не в іншому Git репозиторії, клонуйте вашу копію:

git clone <URLFROMGITHUB>

Перейдіть у щойно створену папку нового репозиторію (в цьому випадку її ім'я 'patchwork').

cd patchwork

Тепер ви маєте на вашому комп'ютері зкопійований репозиторій, і він автоматично під'єднаний до репозиторію на GitHub (той, що у вашому акаунті). Наберіть git remote -v, щоб переконатися, що віддалене з'єднання вже налаштоване.

З'єднайтеся із оригінальним (початковим) репозиторієм

Що якщо оригінальний репозиторій, що ви зкопіювали, змінився? Ви захочете мати можливість зтягнути ці зміни також. Тому давайте додамо нове віддалене з'єднання, цього разу до оригіналу, github.com/jlord/patchwork.

Ви можете назвати це віддалене з'єднання як завгодно, але зазвичай його називають 'upstream'; давайте використаємо саме таку назву.

git remote add upstream https://github.com/jlord/patchwork.git

Аби переконатися, що ви встановили віддалені з'єднання правильно наберіть git remote -v, щоб побачити список усіх адрес. Ви повинні мати віддалене з'єднання 'origin' із адресою вашої копії, а також 'upstream' із адресою оригіналу, яка є вище в цьому кроці
