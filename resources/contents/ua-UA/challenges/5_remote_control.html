З'єднайте ваш локальний репозиторій із віддаленим і додайте ваші зміни до нього.

Віддалені репозиторії

Коли ви додаєте щось до GitHub ця копія буде жити на одному із серверів GitHub'у. Це робить його remote(віддаленим) репозиторієм оскільки воно вже більше не навишому комп'ютері, а на сервері, "віддалене" і де-інде ще. коли ви pushing(додаєте) ваші локальні (на вашому комп'ютері) зміни до нього, ви робите його "сучасним".

Інші можуть завжди отриматі останні зміни вашого проекту шляхом pulling(витягування) ваших змін із віддаленого репозиторію (на їх комп'ютер). Таким чином усі можуть працювати над проектом без необхідності мати доступ до вашого комп'ютеру, де зберігається ваша локальна копія.

Створіть віддалений репозиторій

Ви хочете синхронізувати вашу локальну версію із тією, що на GitHub.com. Спершу створіть новий віддалений репозиторій на GitHub.com

  • Перейдіть на github.com, авторизуйтеся, натисніть '+' у верхньому правому кутку, а потім натисніть 'New repository'.
  • Назвіть його так само, як локальний репозиторій, 'hello-world', і додайте короткий опис.
  • Зробіть його загальнодоступним (public). Це означає, що він буде показаний у вашому публічному профілі.
  • Не ініціалізуйте його із README файлом, оскільки в нас вже є локально файл із ім'ям 'readme.txt'. Це допоміжна опція на GitHub, якщо ви раптом його ще не створили
  • Залишіть '.gitignore' та 'license' опції в 'none'. Ми не будемо їх використовувати в цьому завданні.
  • Натисніть створити репозиторій!

Readmes, .gitignores та ліцензії

Це загальні файли в проектах із відкритим кодом, тому щоб зробити створення проекту простішим GitHub забезбечує можливість створення їх автоматично.

Файл readme пояснює про що цей проект, як його використовувати, і часто, як внести свій внесок (хоча інколи для цих деталей є окремий файл 'CONTRIBUTING.md').

Файл .gitignore — це перелік файлів які Git не повинен відслідковувати, наприклад, файли із паролями!

Файл license — це тип ліцензії, від якою знаходиться проект. Це дозволяє іншим знати як вони можуть використовувати проект. Інформація про види ліцензій: choosealicense.com.

З'єднайте локальний репозиторій із віддаленим

Тепер у вас є пустий репозиторій на GitHub.com. Згорі ви побачите 'Quick Setup' (Швидке налаштування), переконайтеся, що кнопка 'HTTPS' натиснута і скопіюйте адресу — це адреса вашого репозиторію на серверах GitHub.

У вашому терміналі, всередині папки 'hello-world', яку ви ініціалізували як Git репозиторій в попередніх завданнях, ви хочете аби Git запам'ятав адресу віддаленого репозиторію на GitHub. Ви можете мати декілька віддалених репозиторіїв, тому кожен повинен мати ім'я. Головний віддалений репозиторій зазвичай називають origin.

Додати віддалений репозиторій із ім'ям 'origin'до вашого репозиторію:

git remote add origin <URLFROMGITHUB>

Ваш локальний репозиторій тепер знає де саме знаходиться віддалений репозиторій, названий 'origin', на серверах GitHub. Думайте про це, як про додавання ім'я та адреси на "швидкий набір", коли ви хочете відправити туди щось - ви можете.

Користувачам Windows:

Якщо у вас встановлено GitHub Desktop на вашому комп'ютері, то віддалене з'єднання із ім'ям 'origin' вже було створено автоматично. В цьому випадку вам лише потрібно вказати яка саме адреса у 'origin'. Використайте наступну команду замість 'add' згори:

git remote set-url origin <URLFROMGITHUB>

Надішліть вашу роботу у ващ віддалений репозиторій

Далі ви хочете push (надіслати) все, що ви зробили локально, на ваш віддалений репозиторій на GitHub. Це дещо, що ви будете робити досить часто, аби ваша віддалена версія була якомога акиуальніша і відповідала стану вашої локальної версії.

Git має систему гілок, таким чином ви можете працювати над різними частинами проекту у різний час. Ми вивчимо детальніше це пізніше, але за замовчуванням перша гілка має ім'я 'master'. Коли ви надсилаєте (або пізніше зтягуєте) зміни до віддаленого репозиторію ви кажете Git'у ім'я гілки, на яку ви хочете надіслати та ім'я віддаленого репозиторію, де вона знаходиться.

В нашому випадку, ми надсилаємо на гілку 'master' на наш віддалений репоозиторій на GitHub, що має назву 'origin'

git push origin master

Тепер перейдіть на сторінку вашого репозиторію на GitHub та оновіть сторінку. Вау! Все тепер точно таке ж як і локально. Вітаємо із вашим першим публічним репозиторієм!
