Додайте гілку, локально, до вашого зкопійованого репрзиторію аби працювати над вашими змінами

Гілки

Git репозиторії використовують нілки для ізоляції певних частин роботи. Звичайною практикою при роботі над проектом або із іншими - створити гілку, щоб зберегти свої робочі зміни. Таким чином ви зможете продовжувати свою роботу, а головна гілка, зазвичай 'master', залишається стабільною. Коли робота на вашій гілці завершена, ви зливаєте її в гілку 'master'.

Діаграма нижче показує як ви можете створити нову гілку із 'master'у, зробити певну роботу, а потім злити ці зміни назад в 'master'. Ви також можете створити гілку із будь-якої іншої гілки, не обов'язково щоб базовою була саме 'master'.

Для чудової візуалізації як працюють гілки, перегляньте цю сторінку GitHub: guides.github.com/overviews/flow

Сторінки GitHub

GitHub буде автоматично опрацьовувати і робитиме доступним статичні веб файли в гілках із назвою 'gh-pages'. Цей безкоштовний сервіс називається GitHub Pages. Так як зкопійований проект створює веб-сайт, його головна гілка називається 'gh-pages' замість 'master'. Усі репозиторії, що мають гілку 'gh-pages' із веб файлами, можуть бути відвідані в онлайн, знайшовши за наступним URL:

http://githubusername.github.io/repositoryname

Створіть гілку

Коли ви створюєте нову гілку, Git копіює усе із поточної гілки, на якій ви зараз, і розміщує в новій гілці, що ви створюєте.

Допоки все ще всередині вашого локального репозиторію 'patchwork', наберіть git status аби побачити на якій саме гілці ви знаходитесь. Git повинен відповісти, що ви зараз на гілці 'gh-pages'.

Тепер створіть нову гілку і назвіть її "add-<username>", де 'username' це ваше ім'я користувача. Наприклад, "add-jlord". Гілки чутливі до регістру, тому називайте вашу гілку так само, як вказано ваше ім'я користувача на GitHub.

git branch <BRANCHNAME>

Тепер ви маєте гілку із новим ім'ям ідентичну із 'gh-pages'.

Щоб перейти на цю гілку і працювати із нею ви checkout (перемикаєте) гілку. Перейдіть на вашу нову гілку:

git checkout <BRANCHNAME>

Крок: Створіть новий файл

Перейдіть назад у ваш текстовий редактор:

  • Створіть новий файл із ім'ям "add-<USERNAME>.txt", де 'username' це ваше ім'я користувача. Наприклад, "add-jlord.txt".
  • Потім просто запишіть в ньому ваше ім'я користувача на GitHub, і це все. Наприклад, я б набрав 'jlord'.
  • Збережіть файл у папці 'contributors' у репозиторії Patchwork: Patchwork/contributors/add-yourusername.txt
  • Тепер додайте свої зміни до Git (нижче).

Check-in

Зробіть наступні кроки, або зберегти свої зміни до проекту:

git status

git add <contributors/FILENAME>

git commit -m "commit message"

Тепер надішліть свої зміни до вашої копії, 'origin', на GitHub:

git push origin <BRANCHNAME>

Ви намагаєтеся надіслати ваші зміни до репозиторію, в якому не маєте права запису. В даному випадку, швидше за все, ви надсилаєте зміни до оригіналу 'jlord/patchwork'. Переконайтеся, що ви надсилаєте зміни до 'origin' і що він має адресу вашого зкопійованого репозиторію на GitHub. Аби перевірити адреси віддалених з'єднань наберіть git remote -v. Ви повинні мати 'upstream' → 'jlord/patchwork' та 'origin → 'yourusername/patchwork'.

Аби змінити адресу віддаленого з'єднання використайте команду: git remote set-url origin<URLTOYOURFORK>.

Файл НЕ знайдено в папці contributors

Файл, що ви створили, повинен бути розміщений всередині папки 'contributors' в репозиторії 'patchwork'. Якщо ви розмістили його в іншому місці, просто використайте файловий менеджер аби перемістити його. Ви можете перевірити git status знову і ви побачите, що Git бачить ваші зміни. Закомітьте їх усі ("all", -A), використовуючи команди нижче.

git add -A

git commit -m "move file into contributors folder"

Очікуване ім'я гілки: _____

Ім'я гілки повинне бути точною копією вашого ім'я користувача. Щоб змінити ім'я гілки:

git branch -m <NEWBRANCHNAME>
